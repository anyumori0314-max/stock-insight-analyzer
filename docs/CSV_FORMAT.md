# CSV 仕様（日足株価インポート）

`npm run data:import` / `npm run data:daily` が取り込む日足 CSV の仕様です。CSV は
検証後に **SQLite（履歴株価の正本）** へ冪等に保存されます。

> 実際の株価 CSV はリポジトリにコミットしないでください（`.gitignore` で `/data/`,
> `*.local.csv` を無視）。リポジトリにはテスト内のインライン fixture のみを置きます。

## 列

ヘッダー名は **trim 後**に判定します（大文字小文字は区別しません）。

### 必須列

| 列 | 説明 | 検証 |
|----|------|------|
| `ticker` | ティッカー | 既存の ticker validation を再利用（ASCII 英数＋`.`/`-`、最大 10、`AAPL`/`BRK.B` 等）。大文字へ正規化 |
| `date` | 取引日 | 実在する `YYYY-MM-DD` のみ（例: `2026-02-30` は不可） |
| `open` `high` `low` `close` | 始値/高値/安値/終値 | 正の有限数（`NaN`/`Infinity` 不可）。`high ≥ open,close,low` かつ `low ≤ open,close,high` |
| `volume` | 出来高 | 0 以上の安全整数（小数・負数・桁あふれ不可） |

### 任意列

| 列 | 説明 | 検証 |
|----|------|------|
| `adjusted_close` | 調整後終値 | 空、または正の有限数 |
| `currency` | 通貨コード | 空、または 3 文字の英字（`USD` 等）。大文字へ正規化 |

### 未知の列

仕様外の余分なヘッダー（例 `name`, `note`）は **無視**されます（取込結果の
`unknownHeaders` に記録）。データは保存されません。

## 形式ルール

- 文字コードは **UTF-8**。先頭の **UTF-8 BOM** は許容（自動で除去）。
- 引用符付きフィールド（`"..."`）はカンマ・改行・`""`（エスケープ済み引用符）を含められます（RFC 4180 準拠）。
- 空行は無視します。
- **空ファイル** / **データ行なし** / **必須ヘッダー欠落** は取込全体を拒否します。
- **同一ファイル内で `ticker + date` が重複** する行は不正として検出します。
- 上限: 既定で最大 **100,000 行** / **5,000,000 バイト**（`STOCK_IMPORT_MAX_ROWS` /
  `STOCK_IMPORT_MAX_BYTES`）。超過は取込を拒否します。

## 検証と原子性

- **ファイル全体を検証してから** DB トランザクションを開始します。
- 1 行でも不正があれば **何も保存しません**（部分取込は発生しません）。`import_runs` に
  `failed` と安全なエラーサマリ（行番号＋理由）を記録します。
- 同じ CSV を再取込しても重複しません（`ticker + trade_date` で UPSERT）。再取込時の各行は
  insert / update / unchanged として件数が区別されます。

## セキュリティ

- エラーメッセージに値を含める場合も、**CSV 数式インジェクション**（`=`,`+`,`-`,`@` で始まる
  セル）はそのまま再出力せず、先頭に `'` を付けて無害化し、制御文字除去・長さ制限を行います。
- 取込結果・ログには **絶対パスを出しません**（ファイル名のみ）。
- SQL は常にパラメータ化クエリ（`?` プレースホルダ）で、ティッカーや日付を文字列連結しません。

## 例

```csv
ticker,date,open,high,low,close,volume,adjusted_close,currency
AAPL,2026-06-15,195.10,197.40,194.80,196.75,45210000,196.75,USD
AAPL,2026-06-16,196.80,198.20,195.90,197.60,38990000,,USD
MSFT,2026-06-15,420.00,423.10,418.50,422.30,21500000,,USD
```

## 取込コマンド

```bash
# 単一ファイル
npm run data:import -- --file "D:\market-data\AAPL.csv"

# ディレクトリ内の *.csv を一括（再帰なし・シンボリックリンクは追わない）
npm run data:import -- --directory "D:\market-data\daily"
```

終了コード: `0`=成功 / `1`=入力エラー（使い方・ファイル不正・検証失敗）/ `2`=DB エラー。

> `historical` / `hybrid` と データ CLI は **Node ≥ 22.5**（標準の `node:sqlite`）が必要です。
